# 기본 용어 개념 정리 문서

1. 로컬 (Local)  
사전적 의미: (현재 얘기되고 있거나 자신이 살고 있는 특정) 지역의, 현지의
개발분야에서 사용되는 의미: 특정 지역 또는 기기 내에서만 적용되는 것
힌트: "로컬 환경과 서버 환경의 차이 스프링부트", "개발에서 로컬(local) 환경이란 무엇인가", "local 환경에서 애플리케이션 실행 개념"

2. 원격 (Remote)  
사전적 의미: 외진, 외딴 (=isolated)
개발분야에서 사용되는 의미: 네트워크 상의 원격 저장소(GitHub, GitLab 등)에 프로젝트 코드가 저장된 상태
힌트: "원격 서버(remote server) 개념 정리", "개발에서 remote 환경과 local 환경 차이", "git remote repository와 원격 서버 개념"

3. 서버 (Server)  
사전적 의미: 서비스(재화나 무형의 가치) 제공자
개발분야에서 사용되는 의미: 클라이언트를 거쳐 네트워크를 통해 서비스하는 컴퓨터를 의미
힌트: "서버란 무엇인가 백엔드 개발", "Spring Boot 애플리케이션 서버란 무엇인가", "웹 서버와 애플리케이션 서버 차이 개념"

4. 클라이언트 (Client)  
사전적 의미: 전문가의 서비스를 받는 고객, 의뢰인 등의 의미
개발분야에서 사용되는 의미: 데이터 등을 제공받는 쪽을 의미
힌트: "서버 클라이언트 구조란 무엇인가", "클라이언트 요청 서버 응답 HTTP 개념", "웹 클라이언트 브라우저 역할 정리"

5. 빌드 (Build)  
사전적 의미: "건물을 짓다"이다. 그밖에 "만들다" 또는 "체격" 등의 의미
개발분야에서 사용되는 의미: 소스 코드(.java)를 실행할 수 있는 실행파일(.jar)로 만드는 것
힌트: "Gradle 빌드 과정 compileJava processResources classes", "빌드와 실행(run) 차이 정리", "Spring Boot gradle build jar 만들기"

6. 실행 (Run)  
사전적 의미: '실행, 달리다, 달아나다' 등의 뜻
개발분야에서 사용되는 의미: 애플리케이션을 실행하거나 테스트 환경에서 코드를 동작시키는 명령
힌트: "프로그램 실행이란 무엇인가 소프트웨어", "java -jar로 Spring Boot 실행하는 방법", "빌드된 jar 파일 실행과 개발 도중 실행 차이" 

7. 배포 (Deployment)  
사전적 의미: 신문이나 책자 등을 널리 나누어 줌.
개발분야에서 사용되는 의미: 빌드된 코드를 실제 서비스 환경에 적용하는 과정
힌트: "Spring Boot 애플리케이션 EC2 서버 배포 방법", "개발(dev) 운영(prod) 환경 배포 차이", "빌드와 배포 차이 DevOps 관점 정리"  

8. 포트 (Port)  
사전적 의미: 항구를 뜻하는 영어 단어 
개발분야에서 사용되는 의미: 컴퓨터가 각종 신호를 받아들이고 내보낼 수 있도록 연결할 수 있는 연결단자
힌트: "포트 번호란 무엇인가 네트워크", "Spring Boot 서버 포트 변경 application.properties", "포트 충돌 에러 원인과 해결 방법"  

9. IP (Internet Protocol)  
개발분야에서 사용되는 의미: 인터넷 프로토콜(Internet Protocol)의 약자로, 인터넷이 통하는 네트워크에서 어떤 정보를 수신하고 송신하는 통신에 대한 규약을 의미
힌트: "IP 주소란 무엇인가 공인 IP 사설 IP 차이", "서버 접속할 때 IP와 포트 조합 의미", "도메인 이름과 IP 주소 관계 DNS"  

10. JAR (Java ARchive)  
개발분야에서 사용되는 의미: 자바 클래스 파일과, 클래스들이 이용하는 관련 리소스(텍스트, 그림 등) 및 메타데이터를 하나의 파일로 모아서 자바 플랫폼에 응용 소프트웨어나 라이브러리를 배포하기 위한 소프트웨어 패키지 파일 포맷 
힌트: "자바 JAR 파일이란 무엇인가", "Spring Boot jar 패키징과 실행(java -jar) 방법", "bootJar로 생성된 jar 파일 특징"

11. bootRun 명령어 (Gradle BootRun Task)
개발분야에서 사용되는 의미: Gradle을 통해 Spring Boot 애플리케이션을 실행하는 작업(task)
힌트: "Spring Boot gradle bootRun 명령어 역할", "bootRun 개발 환경에서만 사용하는 이유", "EC2 서버에서 ./gradlew bootRun으로 테스트 실행하기", "bootRun과 java -jar 배포 방식 차이" 

## EC2에서 bootRun으로 실행해 보기 (연습 시나리오)

EC2와 같은 연습용 서버에서 Spring Boot 프로젝트를 간단히 실행해 보고 접속이 되는지 확인하는 연습

1단계: EC2 인스턴스 준비  
- AWS 콘솔에 접속하여 EC2 인스턴스를 생성  
- 운영체제는 보통 Ubuntu 계열 리눅스를 사용  
- 인스턴스 보안 그룹에서 8080 포트를 외부에서 접속 가능하도록 설정  

2단계: EC2에 접속  
- 로컬 터미널에서 SSH 명령으로 EC2에 접속  
- 예시: `ssh -i "my-key.pem" ubuntu@EC2_공인_IP`  
- 접속 후 현재 위치를 확인하기 위해 `pwd` 명령을 사용  

3단계: 프로젝트 가져오기  
- EC2 안에서 `git clone 깃허브_레포지토리_URL` 명령으로 GitHub에 올려 둔 프로젝트를 가져오기  
- `cd 프로젝트_디렉터리_이름` 명령으로 프로젝트 디렉터리로 이동  

4단계: 실행 권한 및 Java/Gradle 확인  
- `./gradlew` 파일에 실행 권한을 주기 위해 `chmod +x ./gradlew` 실행  
- `java -version` 명령으로 JDK가 설치되어 있는지 확인  
- 필요하다면 패키지 매니저(예: `sudo apt install openjdk-17-jdk`)로 설치  

5단계: bootRun으로 애플리케이션 실행  
- 프로젝트 디렉터리에서 `./gradlew bootRun` 명령 실행  
- 로그에서 8080 포트로 Tomcat 또는 내장 서버가 시작되었다는 메시지를 확인  
- 필요하다면 `./gradlew bootRun --args='--spring

6단계: 브라우저에서 접속 확인  
- 로컬 브라우저에서 `http://(EC2 공인 IP):(포트 번호)`로 접속  
- 기본 포트는 보통 8080이며, 엔드포인트 예시는 `/hello`, `/api/test` 등 사용  
- 페이지 또는 API 응답이 보이면 bootRun을 사용한 서버 실행이 정상적으로 동작하고 있는 것  

7단계: 서버 종료  
- EC2 터미널에서 `Ctrl + C`를 눌러 실행 중인 애플리케이션을 종료  
- 또는 foreground로 떠 있는 프로세스를 `ps` 명령으로 확인하고, `kill (PID)`로 종료  

### 시나리오 요약

- 로컬에서 실행하던 Spring Boot 프로젝트를  
  연습용 EC2 서버에 git clone으로 가져와서 `./gradlew bootRun`으로 실행해 보고  
  브라우저에서 `http://(EC2 공인 IP):(8080)`로 접속하여 서버와 네트워크 개념을 함께 연습  

- bootRun은 실무 운영 서버 배포에 사용하는 명령이라기보다  
  개발 단계에서 로컬 또는 연습용 서버 환경에서 애플리케이션을 빠르게 실행해 보는 연습용 도구
